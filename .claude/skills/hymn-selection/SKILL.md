# 찬송 선곡 추천 + 선곡표 기록 (광교성서침례교회)

침례교찬송가(1~589) 기준으로 주일예배 찬송을 추천하고, 확인받은 뒤 Notion 선곡표에 기록한다.

- **오전예배:** 메인 2곡(시작 찬송 ①②) + 헌금 1곡 + 마무리 1곡
- **오후예배:** 메인 2곡 + 서브 1곡

핵심 자산은 목사님이 실제로 선곡해 온 **오전예배 155주치 기록(정답지)**과 거기서 도출한 **선곡 논리**다.
목표는 "그럴듯한 찬송"이 아니라 **이 교회·이 목사님의 실제 패턴에 부합하는 추천**이다.

## 경계 (먼저 읽기)

- 목사님의 목양·분별을 대신하지 않는다. 추천은 **보조 제안**이며 최종 선곡은 인도자·목사님의 몫이다.
- 저작권상 **가사 전체를 옮기지 않는다.** 주제·정서·핵심 방향으로 설명한다.
- 타 교단·타 번역본을 비방하지 않는다.

## 절대 규칙 — 근거 (지어내기 금지)

틀린 성구·없는 찬송 하나가 그럴듯한 답보다 훨씬 위험하다. 항상 파일을 **직접 열어** 확인한다.

1. **성경 인용은 흠정역 원문 직접 확인.** 기억 금지, `02_성경_흠정역/`의 책 `.txt`를 연다.
   책→파일은 `02_성경_흠정역/색인.csv`. 파일 안: `제 N 장` 헤딩 아래 `절번호 본문`. 불확실하면 "확인 필요"라 밝힌다.
   흠정역 어휘: 세례→**침례**, 선지자→**대언자**, 성령→**성령님**, 신/구약→**새/옛 상속 언약**, 성경→**성경기록**.
2. **찬송 번호↔제목의 유일 근거는 `03_찬송가/침례교찬송가_색인.csv`.** 색인에 없는 번호는 제시하지 않는다.
   분류·정서는 `03_찬송가/침례교찬송가_가사DB_마스터.md`에서 확인한다(추측 금지).
3. **데이터 품질 주의:** 정답지 CSV 개별 행 중 일부는 목사님 기입오류가 당일 수정됐으나 DB 미반영된
   노이즈일 수 있다. → **한 행에 과의존하지 말고 여러 행의 집계·패턴을 신뢰**한다.

## 추천 절차 (핵심)

먼저 도구로 근거부터 확보한다(반복 조회 자동화):

```bash
python .claude/skills/hymn-selection/scripts/lookup.py --book 로마서 --chapter 8   # 같은/인접 본문 실적
python .claude/skills/hymn-selection/scripts/lookup.py --hymns 59,179,402          # 번호→제목·분류
```

그다음 4단계:

**0) 실적 조회 — 최우선 근거.** `lookup.py --book <책> --chapter <장>`으로 같은/인접 본문의 과거 오전 선곡을 본다.
   연속 강해 중이면 앞뒤 주가 강력한 참고다(노이즈 감안).

**1) 적용 정서 판별 — 가장 중요한 판단.** 흠정역 본문을 직접 읽고 물어라:
   *"이 본문의 사건이 뭔가?"가 아니라 "이 본문이 성도에게 요구하는 **1인칭 응답·정서**가 뭔가?"*
   (확신·성결·헌신·열망·인도받음·위로·감사·회개·순종 …). 앞뒤 구절 맥락까지 본다.

**2) 포지션 지문에 배치**(아래 표). **3) 하나의 정서 아크로 통일** — 곡들이 한 흐름으로 이어지게,
   **성도의 1인칭 응답**을 노래하는 곡으로.

### 오전예배 포지션 지문 (155주 집계)

| 포지션 | 성격 | 기본 후보 풀 | 방식 |
|---|---|---|---|
| **시작 찬송 ①②** (메인 2곡) | 경배로 열고 응답 (인도 21% + 사랑·경배) | 경배: 59·29·49·569·21·18·286 / 인도: 63·49·80·48 / 성령 응답: 177·181 | (선택)경배 오프너 + 그 주 정서 응답곡. **응답곡(오전②)은 성령 계열(177·181)이 상비** — 그 주 '확신곡'(285·292 등)은 오전②가 아니라 **헌금**으로 미루는 것이 실제 패턴 |
| **헌금** | 기도·성찰·간증의 내면 | **179(내 눈을 열어)=기본값** / 346·438·338·285·178·292 | 179 기본에서 그 주 정서로 교체. **확신·승리 주간엔 179→285(내가 확실히 아네)·292(굳게 잡아라)로 교체**(오전에 확신곡 안 쓰고 여기 배치) |
| **마무리** | 헌신·결단의 파송('헌신' 58%) | **402·399·401·396·403**·240·394 | 헌신 파송 풀에서 그 주 색채로 |

> **특별예배(선교주일·감사주일 등)라도 마무리는 응답·헌신 풀을 유지한다.** 그 주제(선교·감사 등)는
> 시작 ②·헌금에 담고, 마무리는 성도의 **응답곡**(402 예수를 위해 사는 것 등)으로 파송하는 것이
> 목사님 실제 패턴이다. 마무리를 주제 분류(선교 등)로 빼는 것은 피한다.

> **오전②·헌금 규칙의 근거(blind eval 2026-07-15):** 확신 주간(롬8·왕상18) 재현에서, '그 주 확신곡'을
> 오전②에 놓으면 빗나가고 **헌금**에 놓아야 실제와 맞았다(오전②는 성령 177 상비). 곡 자체는 맞히는데
> 오전②↔헌금 자리를 바꿔 틀리는 게 최다 에러였다 → 위 두 행에 규칙으로 못박음.

### 오후예배 (메인 2곡 + 서브 1곡)

**⚠ 오후는 과거 선곡 실적이 없다**(정답지 CSV는 오전만). 그래서 오후는 **흠정역 본문 + 가사DB 주제·정서 매칭**으로
추천하며, 오전보다 근거가 약함을 반드시 밝힌다. 오후 설교 본문이 따로 주어지면 그 본문을 기준으로,
없으면 오전 주제의 연장/응답으로 고른다.

- **메인 2곡:** 그 본문의 적용 정서에 맞는 찬양·경배·응답곡(오전 메인과 같은 감각).
- **서브 1곡:** 오후 특성상 조금 더 자유롭게 — 간증·헌신·기도·결단 계열의 마무리성 곡이 자연스럽다.
- 오전 메인과 **곡이 겹치지 않게** 하되, 정서 흐름은 그날 말씀과 통하게 한다.

## 핵심 원리 — 왜 이렇게 고르는가

목사님 선곡의 결정적 축은 **"하나님이 어떤 분인가"(속성 3인칭 찬양)가 아니라 "성도가 어떻게 반응하는가"(1인칭 응답)**다.
곡 1절이 *"나/우리가 ~하리라·붙들자·나아가자"*면 유력 후보고, *"주는 ~하신 분"*류 속성 찬양은 (오프너를 빼면) 잘 안 고른다.

- 검증(왕상 18:1~14, 오바댜의 충성): 채택 = 오전 18·281 / 헌금 292 / 마무리 403 → "환난→확신→승리" 아크.
  본문 '사건'(신실하신 하나님)에 맞춘 속성 찬양(48·49)은 빗나갔다.
- '오전 첫 곡=경배 오프너'는 경향이지 규칙이 아니다. 본문 '책'은 선곡을 거의 결정하지 않는다 —
  **포지션 정서 × 그 주 적용 정서**의 이중 필터로 정해진다.

더 깊은 근거는 `04_예배기록/본문_찬송_연결분석.md`·`선곡패턴_요약.md`·`본문_찬송_enriched.md` 참조.

## 출력 형식 (채팅)

쌍괄식·표 우선·근거 명시·한국어. 곡마다 **왜 이 곡인지** 한 구절로 붙인다. 근거 없는 곡은 넣지 않는다.

```
## 🎵 [예배일] 찬양 선곡

**결론 먼저:** [적용 정서 한 줄 + 정서 아크 요약]
> **근거:** ① 흠정역 [책·장·절] 직접 확인 ② CSV 실적: [오전 같은/인접 본문 실적 유무] ③ 분류는 가사DB.

### 오전예배 (본문: [오전 말씀])
| 포지션 | 번호·제목〔분류〕 | 적용 정서 근거 |

| 시작 ① | … | … |
| 시작 ② | … | … |
| 헌금 | … | … |
| 마무리 | … | … |

### 오후예배 (본문: [오후 말씀]) ※과거 실적 없음 → 주제매칭
| 포지션 | 번호·제목〔분류〕 | 적용 정서 근거 |
|---|---|---|
| 메인 ① | … | … |

**통일된 정서 아크:** […]
```

- 같은/인접 오전 본문 실적이 있으면 표 위에 먼저 제시(1순위). 없으면 "실적 없음 → 주제매칭"이라 밝힌다.
- 추천 끝에 **"이 선곡을 Notion 선곡표에 기록할까요?"**라고 물어 확인을 받는다(다음 절).

## Notion "주일 찬양 선곡표" 기록 (확인 후에만)

사용자가 기록에 동의하면 Notion DB에 한 행을 추가한다. **먼저 같은 예배일 행이 있는지 확인**(중복 방지):
notion-search로 그 날짜/주일을 찾고, 없으면 새로 만든다(있으면 갱신 여부를 묻는다).

- 대상 data source: `collection://d65e43e7-a3f6-4c93-9c22-c950fd378f7d` (BBC KOREA ▸ 주일 찬양 선곡표)
- 도구: `ToolSearch`로 `mcp__claude_ai_Notion__notion-create-pages` 로드 후 사용. 모든 찬송 칸은 **자유 텍스트**라 옵션 걱정 없음.

**속성 매핑** (찬송 칸은 `"번호 제목"` 형식 텍스트, 예: `"402 예수를 위해 사는 것"`):

| 속성 | 값 |
|---|---|
| `주일` (title) | 예: `2026-07-05 주일` (또는 오전 설교제목) |
| `date:예배일:start` | 예배일 `YYYY-MM-DD`, `date:예배일:is_datetime` = 0 |
| `오전 말씀` / `오후 말씀` | 흠정역 본문 (예: `로마서 8:31~39`) |
| `시작 찬송 ①` / `시작 찬송 ②` | 오전 메인 2곡 |
| `헌금 찬송` / `마무리 찬송` | 오전 |
| `오후 찬송 ①` / `오후 찬송 ②` / `오후 찬송 ③` | 오후 메인 2곡 + 서브 1곡 |
| `선곡 근거` | 그날 예배 흐름을 **이야기하듯 자연스러운 2~4문장**으로. (아래 톤 규칙 참고) |
| `상태` | `"검토 중"` (사람 검토 전 초안이므로) |

예배일이 주어지지 않았으면 사용자에게 확인한다(임의로 지어내지 않는다). 날짜 없이 먼저 기록해야 하면
`예배일`을 비워두고 나중에 `update_properties`로 채운다.

**행 생성 후, 페이지 본문(content)에 곡별 근거를 반드시 추가한다**(`notion-update-page`, `insert_content`).
본문에는 **한 곡 한 곡** 왜 골랐는지를 남기되, **사람이 쓴 것처럼 자연스럽게** 쓴다.

> **톤 규칙 (중요):** 이 근거들은 인도자·목사님이 읽는 글이다. `분류 태그〔…〕`·화살표(→)·괄호 나열 같은
> 기계적 조립체를 쓰지 말고, **완결된 문장**으로 그날 예배의 흐름을 이야기하듯 풀어 쓴다.
> 각 곡은 "이 곡은 [본문의 무엇]과 이렇게 이어집니다" 식으로, 가사 한 조각과 말씀을 연결해 한 문장으로 설명한다.

구조 예시(내용은 그날에 맞게):

```
## ☀️ 오전예배 · [본문]

[오늘 오전 말씀이 무엇인지 한 문장 + 어떤 흐름으로 골랐는지 한 문장.]

- **[번호 제목]** — [이 곡을 고른 이유를 자연스러운 한 문장으로.]
- **[번호 제목]** (헌금) — …
- **[번호 제목]** (마무리) — …

## 🌙 오후예배 · [본문]

[오후 말씀 한 문장 + 고른 결 한 문장. 과거 실적 없으면 그 사실을 괄호로 덧붙임.]

- **[번호 제목]** — …
- **[번호 제목]** (서브) — …
```

기록·본문 추가 후 생성된 페이지 URL을 사용자에게 알려준다.
