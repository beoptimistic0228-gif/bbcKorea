# 성서침례교회 사역 보조 (Cowork용)

> **한 줄 요약** — 광교성서침례교회 성도 '낙관'의 성경공부·찬양인도·기도문 작성을
> **킹제임스 흠정역** 안에서 돕기 위한 자료 저장소입니다. Cowork에서 열어 쓰고, git으로 이력을 관리합니다.

이 저장소는 성경 본문 전체를 컨텍스트에 밀어 넣지 않습니다. 대신 **필요한 책 파일만 그때그때 열어 읽는** 구조라, 컨텍스트가 터지지 않고 성구를 정확히 인용할 수 있습니다.

## 핵심 원칙 (근거: `01_지침`)

1. **성구는 절대 지어내지 않는다.** 인용할 땐 반드시 `02_성경_흠정역/`의 해당 책 파일을 **직접 열어** 원문 그대로 가져온다.
2. **흠정역 그대로.** 다른 번역본(개역개정·NIV 등)의 문구를 섞지 않는다. 어휘도 흠정역 표기(침례/대언자/성령님 등)를 쓴다.
3. **찬송가 번호는 색인이 유일 근거.** `03_찬송가/침례교찬송가_색인.csv`에 없는 번호는 제시하지 않는다.
4. **모르면 모른다.** 절 위치·번호가 불확실하면 지어내지 말고 "확인 필요"라고 밝힌다.

## Cowork에서 쓰는 법

1. Cowork(데스크톱)에서 이 폴더를 프로젝트로 연다.
2. 첫 지시로 `01_지침/성서침례교회_프로젝트_지침.md`와 `CLAUDE.md`를 읽게 한다(또는 프로젝트 지침에 붙여넣기).
3. 질문 예: *"요3:16 흠정역 본문 알려줘"* → Claude가 `색인.csv`에서 요한복음 파일을 찾아 `43_요한복음.txt`의 3장 16절만 열어 인용.

> 팁: 책 이름을 명시하면(예: "요한복음 3장") 파일을 더 빨리 정확히 찾습니다.

## 찬송 선곡 스킬

찬송 선곡 추천은 `.claude/skills/hymn-selection/` 스킬이 담당한다 — 설교 본문을 주면 `scripts/lookup.py`로
인접 장의 과거 선곡 실적(`04_예배기록/설교_찬송_실적.csv`)을 조회해 근거 있는 추천을 만든다.
⚠ 실적 데이터는 **오전예배만** 있음(오후예배 실적 수집이 최대 데이터 공백). 스킬 정확도 검증용
`evals/evals.json`(hold-out 3건)은 정의만 되어 있고 아직 실행 전.

## 성경 본문 데이터 출처·검증

- **원본**: 큰 글자 성경 신구약(2025) 킹제임스 흠정역 PDF에서 추출.
- **분할**: 신구약 66권 = 66개 `.txt` (권별 1파일). 각 파일은 `# 책이름` → `제 N 장` → `절번호 본문` 구조.
- **검증 완료**:
  - 66권 장 수·연속성이 알려진 흠정역 장 수와 **완전 일치** (창50·시150·마28·요21·계22 등).
  - 절 수 표본 일치 (창1=31, 시119=176, 요3=36, 마5=48, 계22=21 등).
  - 유명 구절 원문 확인 (요3:16, 창1:1, 시23:1, 롬8:28, 계1:16).
  - 흠정역 어휘 보존 확인 (침례/대언자), 정보상자·부록 누출 없음, 단어중간 공백오류 없음.
- **알려진 사소한 한계**: 본문 소제목 중 상호참조가 없는 일부가 앞 절 끝에 붙어 있을 수 있음(절 본문 자체는 온전).
